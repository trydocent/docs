The Docent API lets external AI agents and developers place trades, check balances, and manage strategies without the mobile app. Each API account gets its own non-custodial wallet powered by [Privy](https://www.privy.io/).

## Base URL

```
https://api.trydocent.com
```

## Authentication

All endpoints (except registration) require a Bearer token:

```bash
curl https://api.trydocent.com/v1/balances \
  -H "Authorization: Bearer dk_live_a1b2c3d4..."
```

API keys use the `dk_live_` prefix. You receive your key exactly once during registration — store it securely.

## Permissions

Each API key has four permission scopes:

| Permission | What it controls |
| --- | --- |
| `read` | Balances, positions, markets, actions, strategies |
| `trade` | Place and cancel orders |
| `transfer` | Move funds between venues |
| `strategies` | Create and cancel DCA/conditional strategies |

All permissions are enabled by default on new keys.

## Rate limits

| Tier | Limit |
| --- | --- |
| Standard | 60 requests/minute |
| Premium | 300 requests/minute |

When rate limited, the response includes a `Retry-After` header and `retryAfter` field (milliseconds).

## How orders work

Orders execute asynchronously via durable workflows:

1. You call `POST /v1/orders/perp` (or `/spot`, `/polymarket`)
2. Docent returns an `actionId` immediately
3. The workflow auto-funds from your unified balance if needed
4. Poll `GET /v1/actions` to check completion

This means your order is guaranteed to execute or fail — even if there's a server restart mid-execution.

## Error format

All errors follow a consistent shape:

```json
{
  "error": "Human-readable message",
  "code": "MACHINE_CODE"
}
```

| Status | Code | Meaning |
| --- | --- | --- |
| 401 | `UNAUTHORIZED` | Missing or invalid API key |
| 403 | `FORBIDDEN` | Key lacks required permission |
| 429 | `RATE_LIMITED` | Too many requests |
| 400 | `VALIDATION_ERROR` | Invalid request body |
| 400 | `ORDER_FAILED` | Order validation failed |
